# S1: Packaging, minimization, tree-shaking

Status: done
Owner: Anthony (via Claude)
Spec: V1_SPEC.md S1 (+ BUILD_PROMPT item 1, audit B1/B2/B3)
Branch: main (work on a feature branch, PR into main)

## Goal

Make the library consumable and tree-shakeable: ship tokens.css, externalize
echarts behind an `echarts/core` wrapper, emit per-module output with
`preserveModules` + `sideEffects`, wire subpath entries, and gate bundle size
with size-limit budgets.

## Acceptance criteria (from V1_SPEC S1)

- 1.1 Per-component cost: a fresh app importing only `Badge` contributes <= 3 KB gz, no chart/table/form code.
- 1.2 Charts isolated under `@sonarmd/ui/charts`; echarts via `echarts/core` with only required charts/components/canvas renderer; `echarts-for-react` deleted; zero echarts bytes when no chart imported.
- 1.3 `preserveModules`; `"sideEffects": ["**/*.css"]`; `echarts` is a peerDependency.
- 1.4 size-limit budgets enforced: primitive <= 3 KB gz, full non-chart surface <= 80 KB gz, charts entry <= 120 KB gz incl. echarts core. (motion/transitions/data budgets land with those workstreams.)
- Subpath entries after S1: `.`, `./charts`, `./tokens`, `./tokens.css`, `./style.css`. (`./motion`, `./transitions`, `./data`, `./testing` are added when those workstreams land.)

## Self-made decisions (record in IMPLEMENTATION_STATUS.md)

- D1. Add a `zIndex` token group to sonarmd-tokens.ts (`dropdown:1000, modal:1100, toast:1200, tooltip:1300`). CSS consumes `--smd-z-*` but no source token existed; completeness check forced this. Priority: minimization is unaffected; needed for tokens.css completeness.
- D2. Drop the UMD single-file output; emit ES + CJS both with `preserveModules`. UMD is incompatible with preserveModules and no script-tag/CDN consumer is in scope. `require` resolves to the CJS preserveModules tree.
- D3. Chart renderer: register `CanvasRenderer` per Criterion 1.2 (current code used SVG). Charts are mocked in snapshots so no snapshot churn.
- D4. The playground "premium" token overrides (radius 10/14, input-height 40, custom border/shadow) are NOT canonicalized; they were playground-only. Canonical tokens.css is generated from sonarmd-tokens.ts as-is. Theming refinement is S2.
- D5. tokens.css generated by a small vite plugin calling a shared pure `buildTokensCss()` (src/tokens/tokensCss.ts); the same function backs the vitest completeness check, so generator and check never drift.
